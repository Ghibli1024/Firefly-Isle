System Map

Firefly-Isle Architecture

One-page oncology treatment timeline builder. React SPA on Cloudflare Pages, Supabase Auth + Postgres + RLS for session-bound records, and a Supabase Edge Function proxy that isolates Gemini access from the browser.

Runtime Flow

  • • Browser enters React SPA through /login, /privacy, /app, or /record/:id.
  • • AppBoot + AuthProvider restore Supabase session before protected routes render.
  • • /app orchestrates extraction input, follow-up rounds, preview, inline edit, and export.
  • • Extraction flow is prompt-driven: free text → JSON parse → normalize → missing-field question.
  • • Timeline rendering consumes a single PatientRecord truth model with ordered treatment lines.

Data / Security

  • • Privacy gate and /privacy page share the same source text from src/lib/privacy.ts.
  • • Frontend never calls Gemini directly; it only hits chat(messages, options) → /llm-proxy.
  • • llm-proxy verifies Supabase bearer JWT before forwarding to Gemini.
  • • Records persist into public.patients and public.treatment_lines.
  • • RLS policies scope read/write/delete to auth.uid()-owned rows, including joined treatment lines.

Delivery

  • • GitHub Actions runs npm ci, lint, type-check, test, and build on main pushes.
  • • Successful CI deploys dist/ to Cloudflare Pages via wrangler-action.
  • • public/_redirects provides SPA fallback routing for deep links.
  • • Runtime env boundary includes VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY, and VITE_SUPABASE_EDGE_FUNCTION_URL.
  • • Gemini API key stays in Supabase secrets, not in the browser bundle.
